Setting `export = none` at the default level means "nothing is exported unless a more specific stanza overrides" — which is exactly what we want for a forwarder app that has nothing to export.

**Extension — append to the script after the existing drift check loop:**

```bash
# HARD-01 extension (Phase 3 Plan 1): also check app.conf header comments
# for v[0-9]+.[0-9]+.[0-9]+ drift (strings-only — missed by the main check
# because the main check only greps `version = X.Y.Z` lines).
for f in "$REPO_ROOT/TA-ODIN/default/app.conf" "$REPO_ROOT/ODIN_app_for_splunk/default/app.conf"; do
    comment_version=$(head -n 10 "$f" 2>/dev/null | grep -Eo '\bv[0-9]+\.[0-9]+\.[0-9]+\b' | head -1)
    if [[ -n "$comment_version" ]] && [[ "${comment_version#v}" != "$canonical" ]]; then
        rel="${f#$REPO_ROOT/}"
        echo "[HARD-01 DRIFT] $rel header comment says $comment_version but canonical is v$canonical"
```
