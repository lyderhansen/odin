### PROD-05: `tools/tests/rollback-dryrun.sh` + `DOCS/ROLLBACK.md` + CI integration

**`tools/tests/rollback-dryrun.sh`** (180 lines, executable, shellcheck-clean): Simulates the production Deployment Server rollback procedure WITHOUT a real Splunk instance per CONTEXT D5. Steps:

1. Captures pre-toggle event count from a live `bash TA-ODIN/bin/odin.sh` invocation
2. Patches a `mktemp` copy of `TA-ODIN/default/inputs.conf` with `sed 's/^disabled = false$/disabled = true/g'` over both `[script://]` stanzas
3. Runs an inline `python3 -c` configparser emulator confirming both stanzas would be skipped by splunkd at next reload
4. Discards the temp patch and captures post-revert event count
5. Asserts `git diff --quiet -- TA-ODIN/default/inputs.conf` (real file MUST be byte-identical to HEAD)
6. Writes `.planning/artifacts/rollback-dryrun.md` with timestamps, counts, commit hash, and PASS/FAIL verdict

`set -euo pipefail`, `mktemp` for working files, `trap cleanup EXIT` for guaranteed temp removal. Inline `python3 -c` (not heredoc) per Phase 4 plan-checker convention for safety under `set -euo pipefail`. Two suppressed shellcheck info-level findings, both with documented rationale (SC2329 cleanup-via-trap; SC2016 markdown-backticks-in-single-quotes).

**Induced-failure detection verified end-to-end:** Replaced `TA-ODIN/bin/odin.sh` with an empty stub → POST_COUNT=0 → VERDICT=FAIL → exit 1; restored → exit 0. Real `inputs.conf` and `odin.sh` both byte-identical to HEAD after the test.

**`.github/workflows/ci.yml`** — added a new "Rollback dry-run guard" step inserted between version-sync and injection-fixtures, mirroring `check-version-sync.sh` wiring. Hard-gate: any non-zero exit from `rollback-dryrun.sh` fails the build. Runs in <2s.
