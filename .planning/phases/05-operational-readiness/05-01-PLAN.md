# Plan 05-01 — Linux Module Standalone-Fallback Hygiene (PROD-07, Wave 0)

## Goal

Bring the 6 Linux module standalone-fallback blocks (gated by `if ! declare -f emit &>/dev/null`) into parity with the orchestrator's canonical guarded `emit()` (`TA-ODIN/bin/odin.sh:55-71`):

1. Version string `1.0.0` (was `2.1.0`, predates v1.0.0 release reset)
2. `ODIN_MAX_EVENTS` guard with `type=truncated` emission when cap is hit

Plus extend `check-version-sync.sh` to grep modules and prevent future drift.

## Why Wave 0

PROD-07 has zero dependencies on other Phase 5 work — it's pure code patches inside `TA-ODIN/bin/modules/` + a script extension. Runs in parallel with Plan 02 (PROD-06 dashboard + PROD-05 rollback) and Plan 03 (PROD-03 runbook + new alert stanza).

## D3 minimal scope (per CONTEXT)

Skipping PROD-07 (d) `_common.sh` consolidation. Modules retain their per-module standalone fallback blocks. Refactor to shared dot-sourced helper deferred to v1.1+ as a focused refactor that touches all 6 modules in a single coordinated change.

## Critical research findings (from 05-RESEARCH.md §6)

- Patch is a byte-for-byte mirror of `TA-ODIN/bin/odin.sh:55-71` adapted for standalone scope (no `safe_val` dependency)
- Standalone-fallback gating (`if ! declare -f emit &>/dev/null`) MUST be preserved — orchestrator-driven runs continue to use the orchestrator's `emit()` and never enter the fallback branch
- `check-version-sync.sh` extension follows existing 2-section pattern (canonical 4 sites, comment drift) — adds a 3rd section grepping modules

## Out of scope

- `modules/_common.sh` consolidation (deferred to v1.1+ per D3)
- Any change to orchestrator-driven behavior (`odin.sh` is unchanged)
- Windows PowerShell module fallback hygiene (`_common.ps1` already has equivalent guard)
- Adding new modules or signal types (Phase 4 / future phases)
