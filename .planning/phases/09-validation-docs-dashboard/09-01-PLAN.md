Insert after the live-capture block, before `exit $fail`:

```bash
# --- Extract type=odin_host_info line from each platform ---
linux_event=$(echo "$linux_out" | grep 'type=odin_host_info' | head -1)
windows_event=$(echo "$windows_out" | grep 'type=odin_host_info' | head -1)

if [[ -z "$linux_event" ]]; then
    echo "[HOST-03 FAIL] Linux orchestrator emitted no type=odin_host_info event"
    fail=1
fi
if [[ -z "$windows_event" ]]; then
    echo "[HOST-03 FAIL] Windows orchestrator emitted no type=odin_host_info event"
    fail=1
fi
[[ $fail -ne 0 ]] && exit $fail

# --- Extract field NAMES (regex: lowercase_word followed by =) ---
# This deliberately matches field NAMES only, not values. Linux os_kernel=5.14
# and Windows os_kernel=10.0.26100 both contribute the field name 'os_kernel'.
linux_fields=$(echo "$linux_event" | grep -oE '[a-z_]+=' | sort -u)
windows_fields=$(echo "$windows_event" | grep -oE '[a-z_]+=' | sort -u)
```

The `grep -oE '[a-z_]+='` extracts every `field_name=` occurrence (the `=` is part of the match for boundary). `sort -u` deduplicates and orders alphabetically for diff stability.

Append to `tools/tests/check-host-info-parity.sh` (after Task 2's extraction block, before final `exit $fail`):

```bash
# --- Diff field sets ---
diff_out=$(diff <(echo "$linux_fields") <(echo "$windows_fields"))
if [[ -z "$diff_out" ]]; then
    field_count=$(echo "$linux_fields" | wc -l | tr -d ' ')
    echo "[HOST-03 PASS] Linux + Windows field-set parity confirmed ($field_count fields each)"
else
    echo "[HOST-03 FAIL] Linux/Windows field-set divergence:"
    echo "$diff_out" | sed 's/^/    /'
    fail=1
fi
```

The `<( )` process substitution feeds the sorted field lists to diff without temp files. `diff` exits 0 (empty output) on identical input, exits 1 (with `<` and `>` diff format) on divergence. The `sed 's/^/    /'` indents the diff output for readable CI logs.

Final field count is captured for the success message — useful sanity (should be 13 for both platforms when D-01..D-07 are honored). If field counts differ in the diff (e.g., one platform missing a field), the FAIL branch shows exactly which fields are extra/missing.
