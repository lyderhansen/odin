Create `tools/tests/windows-fixtures/stubs.ps1` — a single PowerShell file that, when dot-sourced, registers functions shadowing every Windows-only cmdlet any Windows module calls. Per RESEARCH §3, PowerShell resolves functions before cmdlets, so these stubs replace the real cmdlets for the remainder of the session.

**File header comment** must explain: (a) this file is only dot-sourced when `$env:ODIN_TEST_FIXTURE` is set, (b) stubs return `[PSCustomObject]` instances matching real cmdlet output shapes per RESEARCH §1, (c) the `__THROW__` sentinel in services fixture triggers a throw for Dimension 6 fail-soft testing.

**Helper at the top:**
```powershell
function Get-OdinFixturePath {
    param([string]$Module)
    $fixtureDir = $env:ODIN_TEST_FIXTURE
    if (-not $fixtureDir) { return $null }
    if (-not (Test-Path $fixtureDir)) { return $null }
    $path = Join-Path $fixtureDir "$Module.json"
    if (Test-Path $path) { return $path } else { return $null }
}

function Read-OdinFixture {
    param([string]$Module)
    $path = Get-OdinFixturePath -Module $Module
    if ($null -eq $path) { return @() }
    return (Get-Content -Raw -Path $path | ConvertFrom-Json)
}
```

**Functions to register** (use full cmdlet names, no aliases — per RESEARCH §11):

1. `function Get-Service` — reads `services.json`, returns objects with properties Name, DisplayName, Status, StartType. If any record has `Name -eq '__THROW__'` then `throw "Induced failure for fail-soft test"`.

2. `function Get-CimInstance` — accepts `-ClassName` and `-Query`. For `-ClassName Win32_Service`, joins services.json with synthetic StartMode/PathName/StartName. For `-ClassName Win32_Process`, returns processes.json records with CommandLine. Other class names return `@()`.

3. `function Get-NetTCPConnection` — accepts `-State Listen`. Reads ports.json, filters `transport -eq 'tcp'`, returns objects with LocalAddress, LocalPort, State='Listen', OwningProcess.

4. `function Get-NetUDPEndpoint` — reads ports.json, filters `transport -eq 'udp'`, returns objects with LocalAddress, LocalPort, OwningProcess.

5. `function Get-ScheduledTask` — reads scheduled_tasks.json, returns objects with TaskName, TaskPath, State, Author.

6. `function Get-ScheduledTaskInfo` — accepts a TaskName parameter, returns a single object with LastRunTime, LastTaskResult, NextRunTime.

7. `function Get-Process` — reads processes.json, returns objects with Id, Name, Path.

8. `function Get-Volume` — reads mounts.json, returns objects with DriveLetter, FileSystemLabel, FileSystem, Size, SizeRemaining.

9. `function Get-PSDrive` — accepts `-PSProvider FileSystem`, returns pass-through of mounts.json as PSDrive-shaped objects (Name, Used, Free, Root).

10. **Registry stubs** — the tricky ones. `function Get-ChildItem` must intercept only paths starting with `HKLM:\SOFTWARE\...Uninstall`; all other paths must call the real `Microsoft.PowerShell.Management\Get-ChildItem` so tests can still read JSON files. Use:
```powershell
function Get-ChildItem {
    [CmdletBinding()]
    param([Parameter(ValueFromRemainingArguments=$true)]$Args)
    $path = if ($Args.Count -gt 0) { $Args[0] } else { $null }
    if ($path -is [string] -and $path -like 'HKLM:\SOFTWARE*Uninstall*') {
        $packages = Read-OdinFixture -Module 'packages'
        # Return pseudo-registry-key objects; Get-ItemProperty stub handles the read
        return $packages | ForEach-Object {
            [PSCustomObject]@{ PSPath = "HKLM:\Uninstall\$($_.DisplayName)"; __OdinFixtureRecord = $_ }
        }
    }
    return Microsoft.PowerShell.Management\Get-ChildItem @Args
}

function Get-ItemProperty {
    [CmdletBinding()]
    param([Parameter(ValueFromPipeline=$true)]$InputObject, [Parameter(ValueFromRemainingArguments=$true)]$Args)
    process {
        if ($InputObject -and $InputObject.PSObject.Properties['__OdinFixtureRecord']) {
            return $InputObject.__OdinFixtureRecord
        }
        return Microsoft.PowerShell.Management\Get-ItemProperty -InputObject $InputObject @Args
    }
}
```

11. `function Test-Path` for HKLM registry paths — return `$true` for `HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall` and the Wow6432Node variant when fixture is set; pass through to real Test-Path otherwise.

**Absolute prohibitions** (same as production code — stubs are read by planner and checker):
- NO `Invoke-Expression`, NO `Add-Type`, NO `iex`, NO `FromBase64String`, NO `Net.WebClient`, NO `DownloadString`
- NO cmdlet aliases — use `ForEach-Object` not `%`, `Where-Object` not `?`
- NO `Import-Module` of non-built-in modules

**Determinism:** Every stub must return data in a stable, deterministic order. Tests will diff output; non-deterministic ordering breaks Dimension 5.
