**Function 2 — `Format-OdinValue`** — equivalent to Linux `safe_val` in services.sh:29-37. Strip newlines, escape embedded double-quotes, wrap in double-quotes if value contains space, tab, or `=`. Handle `$null` -> empty string.
```powershell
function Format-OdinValue {
    [CmdletBinding()]
    param([Parameter(Mandatory=$false, ValueFromPipeline=$true)][AllowNull()]$Value)
    process {
        if ($null -eq $Value) { return '' }
        $s = [string]$Value
        # Strip CR/LF (replace with single space) so one record = one line
        $s = $s -replace "`r", '' -replace "`n", ' '
        # Escape embedded double quotes
        $s = $s -replace '"', '\"'
        # Wrap if contains whitespace or '='
        if ($s -match '[\s=]') {
            return '"' + $s + '"'
        }
        return $s
    }
}
```

**Function 3 — `Invoke-OdinEmit`** — emits a single event line. Respects `$script:ODIN_MAX_EVENTS`. Uses `Write-Output` only (D5-revised).

```powershell
function Invoke-OdinEmit {
    [CmdletBinding()]
    param([Parameter(Mandatory=$true, Position=0)][string]$Line)
    if ($script:ODIN_EVENTS_TRUNCATED) { return }
    if ($script:ODIN_EVENT_COUNT -ge $script:ODIN_MAX_EVENTS) {
        $script:ODIN_EVENTS_TRUNCATED = $true
        $ts = Get-OdinTimestamp
        $truncLine = "timestamp=$ts hostname=$($env:ODIN_HOSTNAME) os=$($env:ODIN_OS) run_id=$($env:ODIN_RUN_ID) odin_version=$($env:ODIN_VERSION) type=truncated message=`"Event limit reached (max=$($script:ODIN_MAX_EVENTS)). Remaining events suppressed.`""
        Write-Output $truncLine
        return
    }
    $script:ODIN_EVENT_COUNT++
    $ts = Get-OdinTimestamp
    Write-Output "timestamp=$ts hostname=$($env:ODIN_HOSTNAME) os=$($env:ODIN_OS) run_id=$($env:ODIN_RUN_ID) odin_version=$($env:ODIN_VERSION) $Line"
}
```

**Module discovery loop (WIN-01 auto-discovery, WIN-12 fail-soft):**

Use `Get-ChildItem` (full name, NOT `gci`) to enumerate `*.ps1` under the modules directory, excluding `_common.ps1`:
```powershell
$modules_total = 0
$modules_success = 0
$modules_failed = 0

if (-not (Test-Path $ModulesDir)) {
    Invoke-OdinEmit "type=odin_error message=`"Modules directory not found: $ModulesDir`""
} else {
    $moduleFiles = Get-ChildItem -Path $ModulesDir -Filter '*.ps1' -File | Where-Object { $_.Name -ne '_common.ps1' } | Sort-Object Name
    foreach ($moduleFile in $moduleFiles) {
        $modules_total++
        $moduleName = [System.IO.Path]::GetFileNameWithoutExtension($moduleFile.Name)
        Initialize-OdinModuleState

        # D1 CANONICAL PATTERN: Start-Job / Wait-Job -Timeout / Stop-Job / Receive-Job
        $job = $null
        try {
            $job = Start-Job -ScriptBlock {
                param($modulePath, $commonLibPath, $fixtureDir, $hostname, $os, $runId, $version, $maxEvents)
                $env:ODIN_TEST_FIXTURE = $fixtureDir
                $env:ODIN_HOSTNAME = $hostname
                $env:ODIN_OS = $os
                $env:ODIN_RUN_ID = $runId
                $env:ODIN_VERSION = $version
                . $commonLibPath
                $script:ODIN_MAX_EVENTS = $maxEvents
                . $modulePath
            } -ArgumentList $moduleFile.FullName, $CommonLib, $env:ODIN_TEST_FIXTURE, $env:ODIN_HOSTNAME, $env:ODIN_OS, $env:ODIN_RUN_ID, $env:ODIN_VERSION, $script:ODIN_MAX_EVENTS

            $finished = Wait-Job -Job $job -Timeout $ModuleTimeout

            if ($null -eq $finished) {
                # D4: timeout — synthesize exit_code=124
                Stop-Job -Job $job -ErrorAction SilentlyContinue
                Receive-Job -Job $job -ErrorAction SilentlyContinue | ForEach-Object { Write-Output $_ }
                Remove-Job -Job $job -Force -ErrorAction SilentlyContinue
                $modules_failed++
                Invoke-OdinEmit "type=odin_error module=$moduleName exit_code=124 timeout_reason=wait_job_timeout message=`"Module $moduleName exceeded ${ModuleTimeout}s timeout`""
            } else {
                # Module finished within budget — pipe its output through unchanged
                $jobOutput = Receive-Job -Job $job -ErrorAction Continue
                foreach ($line in $jobOutput) { Write-Output $line }
                $jobState = $job.State
                Remove-Job -Job $job -Force -ErrorAction SilentlyContinue
                if ($jobState -eq 'Failed') {
                    $modules_failed++
                    $errMsg = 'module threw an exception'
                    if ($job.ChildJobs -and $job.ChildJobs[0].JobStateInfo.Reason) {
                        $errMsg = $job.ChildJobs[0].JobStateInfo.Reason.Message -replace '"', "'" -replace "`r", '' -replace "`n", ' '
                    }
                    Invoke-OdinEmit "type=odin_error module=$moduleName exit_code=1 error_message=`"$errMsg`""
                } else {
                    $modules_success++
                }
            }
        } catch {
            if ($job) {
                Stop-Job -Job $job -ErrorAction SilentlyContinue
                Remove-Job -Job $job -Force -ErrorAction SilentlyContinue
            }
            $modules_failed++
            $exMsg = ($_.Exception.Message) -replace '"', "'" -replace "`r", '' -replace "`n", ' '
            Invoke-OdinEmit "type=odin_error module=$moduleName exit_code=2 error_message=`"$exMsg`""
        }
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Splunk UF process → odin.path → powershell.exe | UF reads .path as text; attacker with write access to odin.path could redirect the interpreter. Mitigation: file lives under Splunk-controlled `%SPLUNK_HOME%\etc\apps\TA-ODIN\bin\` with Splunk service ACLs. |
| Parent pwsh process → Start-Job child | Child job runs in a separate PowerShell process. No shared variable state by design. Arguments passed via `-ArgumentList` are serialized across the boundary. |
| `$env:ODIN_*` vars → Invoke-OdinEmit output | Env vars flow into the emit prelude (`hostname=$env:ODIN_HOSTNAME ...`). A tampered env var with shell metacharacters could confuse Splunk's KV_MODE parser. Mitigation: Format-OdinValue quotes and escapes values with whitespace or `=`. |
| `$env:ODIN_TEST_FIXTURE` → dot-source of stubs.ps1 | Setting this env var triggers dot-sourcing of arbitrary stubs.ps1 content. In production Splunk UF, this env var is NEVER set. |
